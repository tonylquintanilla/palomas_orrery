One line: a top-bar Reset button, behind a confirm dialog, that returns the GUI
to its STARTUP state -- now-dated, nothing selected, Sun/Auto, default scalars.

Decisions (settled, do not re-litigate without a demonstrated failure):
- Placement: top bar next to the Vernal Eq button (`vernal_equinox_button`, 8500).
- Confirm dialog YES ("Clear all selections and reset to defaults?") -- the pause
  is intentional; preserving anything is the user's manual job.
- Scope = FULL startup state, not a curated subset: date->now; every checkbutton
  cleared (bodies, shells, celestial, comets, spacecraft); center->Sun;
  scale->Auto; plot scalars (days-to-plot, animation step, trajectory interval,
  camera-tracking, encounter date-override) -> startup values.
- Architecture = OPTION A: one loop over the `objects` list + clear the handful of
  shell/celestial/special var groups + reset the scalars, with a one-line
  `# add new checkbox families here` comment. NOT a registry/grouped-list refactor

  demonstrated partial-reset failure proves it load-bearing (bar-for-modification).

  Reset cannot close already-open Plotly browser tabs or wipe console scrollback
  -- those live outside the app. "Clean slate for the next plot," not "erase past
  output." Accepted; out of scope.

family is a partial reset -- looks clean, isn't (same failure class as the
encounter stickiness and parallel-pipeline drift). So map every selection-var
family before writing the clear.
